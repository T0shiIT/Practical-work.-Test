# Практическая работа. Тест

1.Для чего на практике чаще всего используются файлы .env?
>Ответ: c. Для хранения секретов и конфигурационных параметров, загружаемых в переменные окружения

2.Язык С++. В форме main с параметрами, что хранит параметр argv?
>Ответ: a. Массив строк аргументов, включая имя программы

3.Какой ключ g++ используется для остановки компиляции после этапа препроцессирования и вывода результата на экран (или в файл)?
>Ответ: d. g++ -E

4.Какое значение ключевого слова inline для функции в С++ с точки зрения компоновщика (линковки)?
>Ответ: b. Разрешить множественное определение функции в разных единицах трансляции при условии, что все определения идентичны

5.Что означает переменная $< в рецепте Makefile?
>Ответ: a. Первая зависимость

6.Какая команда используется в CMake для добавления пути к каталогу с заглавочными файлами для конкретной цели?
>Ответ: c. target_include_ directories()

7.Каково назначение ключевого слова extern при объявлении переменной или функции в C++?
>Ответ: d. Указать, что переменная или функция определена в другой еденице трансляции (файле)

8.Какой ключ g++ используется для компиляции исходного файла в объектный файл (.о) без этапа линковки?
>Ответ: b. g++ -C

9.Какая директива препроцессора удаляет ранее определенный макрос?
>Ответ: a. #undef

10.Какой символ в начале строки рецепта Makefile подавляет вывод самой команды перед ее выполнением?
>Ответ: d. @

11.Какой будет результат после препроцессирования?
```
#define true false
#define false true

if (true) cout << 1;
else cout << 2;
if (false) cout << 3;
else cout <<4;
```
>Ответ: e. 23

12.Какой файл является обязательным для модуля Go (начиная с версии 1.11)?
Ответ: b. go.mod

13.Какой ключ g++ повзволяет получить ассемблерный код после этапа компиляции, но до ассемблирования?
Ответ: d. g++ -S

14.Что произойдет при запуске make clean, если в каталоге существует файл с именем clean и цель не объявлена как .PHONY?
```

      rm -f main
```
Ответ: a. make выдаст сообщение, что цель актуальна

15.Что произойдет при попытке переопределить существующий макрос без его предварительного удаления?
Ответ: d. Новое определение заменит старое без предупреждения

16.Для чего используется конструкция extern "C" в С++ коде?
Ответ: b. Чтобы отключить name mangling для функции

17. Как в Linux передать вывод одной программы на ввод другой?
Ответ: e. program1 | program2

18. Какую команду необходимо выполнить в терминале (при настройке проекта), чтобы vcpkg создал файл манифеста?
Ответ: e. vcpkg new --application

19.Что такое статическая библиотека в контексте сборки программ?
Ответ: a. Архивный файл (например a), содержащий скомпилированые объектные файлы, которые включаются в исполняемый файл на этапе линковки

20.Какой ключ g++ позволяет добавить каталог к списку путей для поиска заголовочных файлов?
Ответ: a. -I

21.Каково назначение ключевого слова static применительно к глобальной функции или переменной в C++ (в контексте видимости за пределами файла)?
Ответ: a. Сделать функцию или переменную видимой только в пределах того файла (еденицы трансляции), где она объявлена

22.Язык Go. Как в программе получить все аргументы командной строки, кроме имени самой программы?
Ответ: d. os.Args[1:]

23.Как в Linux передать данные из файла на стандартный ввод программе?
Ответ: c. program < file.txt

24.Язык С++. Чем отличается объявление функции с extern от объявления без него (например extern void foo(); vs void foo();)?
Ответ: d. Ничем, для функции extern необязателен, так как они по умолчанию имеют внешнее связывание

25.Чем отличается обработка препоцессором директив #include "file.h" и #include <file.h>?
Ответ: e. #include "file.h" ищет файл сначала в текущем каталоге, затем системных, а #include <file.h> - только в системных

26.Какой плюч g++ используется для создания динамической библиотеки?
Ответ: b. -shared

27.Какие ключи g++ используется для линковки со статической библиотекой libmylib.a, находящейся в текущем каталоге?
Ответ: e. g++ -L. -lmylib

28.Какой ключ g++ позволяет явно указать имя выходного файла (исполняемого файла или библиотеки)?
Ответ: d. -o

29.Какая проблема характерна для классического (глобального) режима vcpkg, но решается в режиме манифеста?
Ответ: b. Возможность конфликта версий, если разным проектам нужны разные версии одной библиотеки

Ответ: d. GetEnv() возвращает пустую строку если переменной нет, LookupEnv() - значение и флаг существования

31.Какая автоматическая переменная в Makefile содержит список всех зависимостей текущей цели?
Ответ: e. $^

32.Какой параметр CMake позволяет указать тип сборки (Debug/Release) для одноконфигурационных генераторов?

33.Язык С++. В форме main с параметрами, что хранит параметр argc?
Ответ: b. Количество переданных аргументов, включая имя программы

34.Какой механизм позволяет CMake находить библиотеки, установленные в системе?

39.Какие каталоги в основном хранилище vcpkg можно периодически удалять для экономии места на диске, так как они содержат кэш и промежуточные файлы сборки?

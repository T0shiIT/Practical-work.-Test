# Практическая работа. Тест

1.Для чего на практике чаще всего используются файлы .env?
Ответ: c. Для хранения секретов и конфигурационных параметров, загружаемых в переменные окружения

2.Язык С++. В форме main с параметрами, что хранит параметр argv?
Ответ: a. Массив строк аргументов, включая имя программы

3.Какой ключ g++ используется для остановки компиляции после этапа препроцессирования и вывода результата на экран (или в файл)?
Ответ: d. g++ -E

4.Какое значение ключевого слова inline для функции в С++ с точки зрения компоновщика (линковки)?
Ответ: b. Разрешить множественное определение функции в разных единицах трансляции при условии, что все определения идентичны

5.Что означает переменная $< в рецепте Makefile?
Ответ: a. Первая зависимость

6.Какая команда используется в CMake для добавления пути к каталогу с заглавочными файлами для конкретной цели?
Ответ: c. target_include_ directories()

7.Каково назначение ключевого слова extern при объявлении переменной или функции в C++?
Ответ: d. Указать, что переменная или функция определена в другой еденице трансляции (файле)

8.Какой ключ g++ используется для компиляции исходного файла в объектный файл (.о) без этапа линковки?
Ответ: b. g++ -C

9.Какая директива препроцессора удаляет ранее определенный макрос?
Ответ: a. #undef

10.Какой символ в начале строки рецепта Makefile подавляет вывод самой команды перед ее выполнением?
Ответ: d. @

11.Какой будет результат после препроцессирования?

```
#define true false
#define false true

if (true) cout << 1;
else cout << 2;
if (false) cout << 3;
else cout <<4;
```

Ответ: e. 23
